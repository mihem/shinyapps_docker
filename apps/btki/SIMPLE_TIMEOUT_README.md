# 简单会话超时机制

## 功能说明

实现了一个简单的会话超时机制：
- **30分钟**无操作后，触发警告
- 弹出**5分钟倒计时**通知
- 倒计时结束后自动**清理内存并断开连接**
- 倒计时期间任何用户操作都会**取消警告**并重新计时

## 文件说明

### 1. 前端 JavaScript
**文件**: `www/js/simple_timeout.js`
- 监听用户活动（鼠标、键盘、滚动、触摸）
- 30分钟无活动后显示警告
- 5分钟倒计时（**前端直接更新DOM，无闪烁**）
- 与服务器通信

### 2. 后端处理器
**文件**: `server/timeout_handler.R`
- 接收前端超时事件
- 显示/更新通知
- 执行内存清理
- 断开会话连接

### 3. 集成位置
- `ui/ui_main.R`: 引入 JavaScript 文件
- `server/server_main.R`: 加载超时处理器

## 参数调整

如需修改超时时间，编辑 `www/js/simple_timeout.js`:

```javascript
var IDLE_TIMEOUT = 30 * 60 * 1000;  // 30分钟无操作（可修改）
var COUNTDOWN_TIME = 5 * 60 * 1000; // 5分钟倒计时（可修改）
```

## 用户体验

1. 用户正常使用应用（任何鼠标、键盘操作都会重置计时）
2. 30分钟无操作后，出现黄色警告通知："由于长时间无操作，系统将在 X分X秒 后自动断开..."
3. 通知每秒更新倒计时
4. 用户可以：
   - **点击任意位置**继续使用（警告消失，重新开始30分钟计时）
   - **不操作**，5分钟后自动断开
5. 断开时显示红色通知："会话已超时，正在断开连接..."
6. 自动清理内存并关闭会话

## 优势

✅ **简单**: 只有2个文件，逻辑清晰
✅ **轻量**: 无复杂依赖
✅ **流畅**: 倒计时在前端更新，无闪烁
✅ **友好**: 有充足的警告时间
✅ **可靠**: 自动清理内存，避免资源泄漏
✅ **灵活**: 易于调整超时参数## 注意事项

- 倒计时期间，用户**无需手动点击"继续"按钮**，任何操作都会取消警告
- 断开后用户需要**重新登录**
- 内存清理调用 `gc()`，适用于大部分场景
