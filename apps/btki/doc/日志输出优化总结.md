# 应用日志输出优化总结

## 问题描述
之前的应用启动时会输出大量不必要的调试信息，包括：
- WebSocket 通信日志 (SEND/RECV 消息)
- 异步加载器的详细日志 ([ASYNC START], [ASYNC LOADED] 等)
- 编排器的模块初始化日志 ([ORCHESTRATOR] 消息)
- Shiny 的调试跟踪信息

## 解决方案

### 1. 异步加载器改进 (`R/async_loader.R`)
- 添加了 `verbose` 参数到 `create_async_loader()` 函数
- 默认创建安静模式的加载器 `create_default_loader(verbose = FALSE)`
- 保留详细模式选项供调试使用

### 2. 编排器改进 (`R/orchestrator.R`)
- 添加了 `verbose` 参数到 `create_orchestrator()` 函数
- 默认使用安静模式 `setup_orchestrator(..., verbose = FALSE)`
- 所有日志输出都受 `verbose` 控制

### 3. 应用启动配置改进 (`R/run_app.R`)
- 修改 `run_app_dev()` 函数默认禁用调试模式
- 添加 `run_app_dev_verbose()` 函数供需要详细日志时使用
- 禁用了 Shiny 的调试跟踪选项

### 4. 全局配置优化 (`global.R`)
- 使用 `suppressPackageStartupMessages()` 减少库加载信息
- 设置 `options(shiny.trace = FALSE)` 等禁用详细输出

### 5. 服务器配置优化 (`R/app_server.R`)
- 创建安静模式的异步加载器和编排器
- 保留重要的系统消息，隐藏详细的调试日志

## 使用方式

### 默认安静模式（推荐）
```r
source('app.R')  # 现在会使用安静模式
```

### 详细调试模式（开发时使用）
修改 `app.R` 中的调用：
```r
# 改为详细模式
run_app_dev_verbose(port = port)
```

或者在 R 中：
```r
source("R/run_app.R")
app <- run_app_dev_verbose(port = 14242)
```

### 异步加载器详细模式
```r
loader <- create_default_loader(verbose = TRUE)
```

### 编排器详细模式
```r
orchestrator <- setup_orchestrator(session, async_loader, module_specs, verbose = TRUE)
```

## 优点
1. **用户体验更好** - 减少了干扰性的技术信息
2. **保持功能完整** - 所有功能都正常工作，只是日志更简洁
3. **调试友好** - 仍然可以通过参数启用详细模式
4. **向后兼容** - 现有代码无需修改，只是默认行为更安静

## 文件修改总结
- `R/async_loader.R` - 添加 verbose 参数控制
- `R/orchestrator.R` - 添加 verbose 参数控制
- `R/run_app.R` - 修改默认调试模式
- `R/app_server.R` - 使用安静模式组件
- `global.R` - 减少库加载信息输出

现在应用启动时会显示简洁、专业的输出，同时保留了必要的状态信息供用户了解加载进度。
