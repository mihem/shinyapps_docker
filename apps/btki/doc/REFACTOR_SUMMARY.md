## 📊 重构前后对比

### 代码复杂度
```
旧架构:
- server_main.R: 655行 (数据加载+模块管理+业务逻辑混在一起)
- 逻辑重复，难以维护

新架构:
- app_server.R: ~80行 (清晰的分层，各司其职)
- 模块化设计，易于扩展
```

### 文件组织
```
旧结构:                     新结构:
====================================
server/server_main.R       R/app_server.R      (精简)
(无中央注册)                R/data_registry.R   (集中管理)
(散布的异步逻辑)            R/async_loader.R    (统一调度)
(隐式模块依赖)              R/module_specs.R    (声明式)
(手动协调)                  R/orchestrator.R    (自动编排)
app.R                      R/run_app.R         (标准化启动)
```

## 🚀 新架构核心优势

### 1. 性能优化
- ✅ **首屏加载速度提升**: 小文件同步加载，大文件异步后台处理
- ✅ **并发控制**: 避免IO竞争，最大化资源利用
- ✅ **优先级调度**: 重要资源优先加载
- ✅ **智能缓存**: 减少重复加载，降低内存占用

### 2. 代码质量
- ✅ **关注点分离**: 数据、UI、业务逻辑明确分工

- ✅ **可测试性**: 每个组件可独立测试
- ✅ **可维护性**: 添加新功能只需修改配置

### 3. 开发体验
- ✅ **实时状态监控**: 清楚了解加载进度
- ✅ **详细日志**: 方便调试和性能分析
- ✅ **错误处理**: 优雅的失败恢复机制
- ✅ **热重载**: 开发时快速迭代

## 📁 新增的核心文件

### R/data_registry.R
- 🎯 **作用**: 集中管理所有数据资源配置
- 📝 **包含**: 路径、大小、优先级、加载函数、依赖关系
- 🔧 **扩展**: 添加新数据源只需在此配置

### R/async_loader.R
- 🎯 **作用**: 智能的异步数据加载调度器
- 📝 **功能**: 并发控制、优先级队列、进度跟踪、错误处理
- 🔧 **配置**: 可调整并发数、超时时间等参数

### R/module_specs.R
- 🎯 **作用**: 声明式的模块配置中心
- 📝 **定义**: 每个模块的资源依赖、准备条件、初始化逻辑
- 🔧 **类型**: immediate(立即) vs lazy(延迟) 模块

### R/orchestrator.R
- 🎯 **作用**: 自动协调数据加载和模块初始化
- 📝 **功能**: Tab监听、依赖检查、智能调度、状态管理
- 🔧 **智能**: 资源就绪时自动初始化对应模块

### R/app_server.R
- 🎯 **作用**: 精简的主server逻辑
- 📝 **职责**: 系统初始化、编排器设置、状态展示
- 🔧 **简洁**: 从655行减少到~80行

### R/run_app.R
- 🎯 **作用**: 标准化的应用启动接口
- 📝 **模式**: 开发模式(调试) vs 生产模式(性能)
- 🔧 **便利**: 一行代码启动应用

## 🎮 如何使用新架构

### 立即开始
```r
# 方式1: 直接运行
source("app.R")

# 方式2: 使用新架构接口
source("R/run_app.R")
run_app_dev(port = 14242)  # 开发模式
# run_app_production()     # 生产模式
```

### 监控和调试
```r
# 查看实时状态
# 应用界面会显示: "模块: 8/15 已初始化, 3 待处理 | 资源: 12 已加载, 2 加载中"

# 编程式查询状态
loader$get_status()
orchestrator$generate_status_report()
```

### 添加新功能

#### 添加新数据源
```r
# 在 R/data_registry.R 中添加
data_resources$my_new_data <- list(
  path = "data/my_analysis.qs",
  type = "analysis",
  blocking = FALSE,           # 异步加载
  priority = 30,             # 中等优先级
  description = "我的新分析",
  load_fn = function(path) qread(path)
)
```

#### 添加新模块
```r
# 在 R/module_specs.R 中添加
module_specs$my_new_module <- list(
  type = "lazy",
  resources = c("my_new_data"),
  tab_name = "my_analysis",
  description = "我的新分析模块",
  ready_fn = function(loader) loader$is_loaded("my_new_data"),
  init_fn = function(id, resources) {
    my_analysis_Server(id, data = resources$my_new_data)
  }
)
```

## 📈 性能提升预期

### 首屏加载时间
- **旧方式**: 所有数据文件竞争加载，首屏等待所有futures
